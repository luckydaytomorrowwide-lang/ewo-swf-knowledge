# Reference Map

このドキュメントは
EWO Factory の参照関係を定義する。

---

# 参照順

作業者およびAIは
次の順序でGitHubを参照する。

1. spec
2. decisions
3. examples
4. templates
5. checklists

spec が最優先である。

---

# spec

仕様の正本（SSOT）。

EWO生成仕様
運用ルール
リポジトリ構造
PDCA工程
テスト構造

を定義する。

---

# decisions

設計判断の記録。

次の場合に記録する。

* 設計判断
* 仕様変更
* 運用ルール変更

---

# examples

EWO実行の再現ケース。

examples は
EWO動作の確認および参照例として使用する。

---

# templates

作業テンプレート。

例

* Pull Request テンプレート
* 作業テンプレート

---

# checklists

作業チェックリスト。

作業開始前
作業完了前
などの確認項目を定義する。
