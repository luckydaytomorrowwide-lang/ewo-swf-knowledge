# Checklist: CHECK工程（期待トレース vs 実ログ 比較）

## 0. 資材取得（GitHub）
- [ ] EWO（実行対象のjson）を取得できる（blob URLを記録）
- [ ] input.json を取得できる（blob URLを記録）
- [ ] table.json を取得できる（blob URLを記録）
- [ ] log.json（対象ログ。再現試験なら複数）を取得できる（blob URLを記録）
- [ ] 使用AC一覧を特定できる（EWOから抽出）
- [ ] 各ACのソース（blob URL）を取得できる（可能なら commit SHA も記録）
  - 取得できない場合：
    - [ ] 不足資材を依頼した（依頼先/依頼内容を記録）
    - [ ] 取得できる範囲で部分解析として進める

## 1. 期待トレース生成（ACソース準拠シミュレート）
- [ ] 各ACについて、入力（deps解決後）を列挙した
- [ ] Search系ACについて、期待される検索条件（parentId/nCat/uCat/key等）を明文化した
- [ ] table.json 上で該当条件に一致する行（line_id）を算出した
- [ ] FirstRow等の選択ロジック（先頭/ソート規則）が実装準拠であることを明記した
- [ ] 期待トレースをステップ番号付きで記録した

## 2. 実トレース抽出（ログ）
- [ ] log から各ACの入力/SQL/出力（件数・IDs）を抽出した
- [ ] 期待トレースのステップと対応付けた（ステップ番号で紐付け）

## 3. 差分判定
- [ ] 期待 vs 実 の差分をステップ単位で列挙した
- [ ] 差分を分類した（入力欠落 / 条件生成差 / ソート差 / null扱い差 / 資材参照違い疑い）
- [ ] 「あり得ない」判定の有無を決定した（理由を明記）

## 4. 結論（CHECKレポートに必須）
- [ ] 判定：OK / 再現試験要 / 詳細解析要 を明記した
- [ ] 再現試験要の場合：同一資材での再実行依頼内容（対象/期待/提出物）を記載した
- [ ] 詳細解析要の場合：次に確認する論点（AC分岐/条件生成/依存先）を列挙した
