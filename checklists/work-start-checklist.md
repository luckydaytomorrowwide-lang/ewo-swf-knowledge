# 作業開始チェックリスト（Plan固定）

## 1. 目的

本チェックリストは、

作業開始前に必ず過去の知識を参照し、
同じ失敗を繰り返さないための強制構造である。

Planを省略する自由を排除する。

---

# 2. 使用方法

- Issue作成時に本チェックリストをコピーして記載する
- PR作成時にも参照確認を再記載する
- 未確認項目がある場合は作業開始不可

# 9. 本チェックの意図

本チェックは「確認のため」ではない。

- 再発防止のため
- 議論の再発防止のため
- 属人判断排除のため

Planを制度化しなければ、
PDCAは回らない。

本チェックはそのための強制装置である。
