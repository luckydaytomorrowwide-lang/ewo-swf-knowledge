# Decision: 参照場所固定

## 決定日
2026-03-04

## 決定内容

- repository.md にリポジトリURLを明記する。
- reference-map.md に参照ブランチとパスを明記する。
- activities / workflows の正確な場所をspecに固定する。

## 背景

- URLを都度確認する必要があった。
- 参照先が曖昧で判断が停止する場面があった。

## 他の選択肢

- チャット内で都度確認する。
- ドキュメント化しない。

## 採用理由

- 参照迷子を防止できる。
- 判断停止リスクを減らせる。

## 今後の判断基準

- 置き場所が決まったものは必ずspecに書く。
- specにない参照は未確定とみなす。
